# Day 22 Notes

## What is the difference between git add and git commit?

git add moves changes to the staging area.
git commit saves staged changes permanently in repository history.

---

## What does the staging area do?

The staging area allows us to review and select changes before committing them.

---

## What information does git log show?

git log shows commit history including commit ID, author, date, and commit message.

---

## What is the .git folder?

The .git folder contains all repository data including commits, branches, logs, and configuration.

If deleted, Git can no longer track the project.

---

## Difference between Working Directory, Staging Area and Repository?

Working Directory:
Current files being edited.

Staging Area:
Prepared changes waiting for commit.

Repository:
Permanent commit history stored by Git.
